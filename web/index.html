What Is This?

This is a Computational Amicus Explorer — an interactive tool that visualizes how a formal mathematical proof system (ACL2) analyzes the constitutional implications of the Federal SAVE Act (H.R. 22).

What does ACL2 do here?

ACL2 is a theorem prover used in hardware and software verification. In this project, it models the SAVE Act's voter registration process as a state machine and derives conditional conclusions about constitutional conflicts — given a set of disclosed assumptions.

ACL2 does not decide the case. It proves conditional consequences from disclosed assumptions. Humans still decide empirical truth, doctrinal soundness, and which interpretation a court should adopt.

How to Navigate

1
Read the graph top-to-bottom. The six layers flow from source materials down through formalization, the executable model, derivations, theorems, and final conclusions.
2
Click any node to see its details in the right panel — source references, the ACL2 event name, which book it lives in, and what it depends on.
3
Toggle assumptions in the left panel. Uncheck an assumption and watch dependent nodes dim and conclusion status badges change from Supported to Unsupported.
4
Try a scenario: Uncheck all challenger assumptions and keep only government assumptions. The Government "NO CONFLICT" conclusion will show Supported while the Challenger conclusion shows Unsupported.

Key Badges

0 Axioms — Pure structural theorem, proved without any assumptions.
High Risk — Contestable empirical premise that influences the outcome.
Trusted Base — Accepted as true but not proved by ACL2.

Final conclusions are conditional, not judicial determinations. This tool visualizes certified proof dependencies — it does not perform live theorem proving or decide legal outcomes.
