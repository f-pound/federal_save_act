# Agent Task: Build a Dynamic Visual Explorer for the Federal SAVE Act Computational Amicus Brief

Repository:

```text
https://github.com/f-pound/federal_save_act
```

## Objective

Add a lightweight, portable, repo-local web explorer that lets users visually navigate the Federal SAVE Act proof-dependency graph.

The goal is **not** to build a complex web application, legal AI engine, or live theorem prover.

The goal is to create a simple dynamic visual enhancement that helps users understand:

```text
sources
-> ACE-style controlled-English propositions
-> ACL2 predicate targets
-> assumptions / trusted base
-> executable process model
-> lemmas and theorems
-> challenger and government conditional conclusions
```

The explorer should allow users to toggle hypotheticals and see which assumptions influence which conclusions.

This should be implemented as a **static-first browser app** served by a simple Python web server.

No Flask, Django, React build system, database, npm pipeline, or live ACL2 execution should be required.

---

## Conceptual Framing

This feature should support the project's **A Computational Amicus Brief** concept.

The explorer should make clear:

```text
ACL2 does not decide the case.
ACL2 proves conditional consequences from disclosed assumptions.
The web explorer visualizes those dependencies.
Humans still decide empirical truth, doctrinal soundness, and which interpretation a court adopts.
```

The explorer is a **proof-dependency and hypothetical-assumption viewer**, not a legal-decision engine.

---

## Design Goal

A user should be able to open the tool and answer questions like:

```text
What happens if I reject the challenger's material-burden assumption?

What happens if I accept the government's "burden not severe" assumption?

Which conclusions depend on the mandatory alternative-process reading?

Which conclusions depend on the discretionary alternative-process reading?

Which theorems are axiom-free structural ACL2 proofs?

Which assumptions are source-traced but not proved?

Which pieces of statutory text map to which ACL2 predicates?

Where does valid-regulationp enter the final outcome?
```

## Required Graph Content

The initial graph should include these major conceptual layers.

### Layer 1: Sources and Traceability

Include:

```text
Legal / doctrinal sources
  - H.R. 22 / SAVE Act text
  - NVRA baseline
  - Constitutional provisions: 5th, 14th, 17th, 24th Amendments
  - Case law: Crawford, Anderson-Burdick, Fish v. Kobach, etc.

Empirical / record sources
  - Fish v. Kobach record
  - roughly 18,000 applicants unable to complete registration
  - Crawford burden discussion
  - material acquisition burden
  - government counter-record: burden not severe / adequate alternative process

Traceability artifacts
  - ACE-style controlled-English propositions
  - source_manifest.json
  - clause_trace.csv
  - proof logs and reports
```

Important correction:

```text
Do not say the model relies on the 15th or 26th Amendments as proof-driving constitutional provisions.
The model uses 5th, 14th, 17th, and 24th Amendment material.
15th and 26th may appear as contextual material only if clearly labeled.
```

### Layer 2: Formalization and Assumptions

Include:

```text
Source-traced text facts
Scenario facts
High-risk empirical assumptions
Interpretive assumptions
Doctrinal assumptions
Bridge rules
Executable definitions
```

High-risk empirical assumptions should be visually prominent:

```text
challenger-scenario-no-fault
  citizen lacks documents through no fault

challenger-scenario-material-burden
  obtaining documents is materially burdensome

government-burden-not-severe
  burden is not severe / adequate alternative process exists
```

These should be visually marked as:

```text
High-risk trusted base
Contestable
Not proved by ACL2
Outcome-influencing
```

### Layer 3: Executable ACL2 Model

Include:

```text
Registration state machine
Document model
Alternative process model
Burden model
Legal predicates
Existential modeling
```

The registration state machine must include the `:alt-pending` state.

Show the alternative path roughly as:

```text
:submitted
  -> :alt-pending
  -> :alt-approved
  -> :registered
```

and the denial path roughly as:

```text
:submitted
  -> :alt-pending
  -> :alt-denied
  -> :denied
```

Do not omit `:alt-pending`.

### Layer 4: Derivations and Intermediate Results

Include:

```text
Denial derivation
Lacks-documents derivation
Document-unavailability derivation
Alternative-process derivation
Burden derivation
Severe-burden derivation
Interpretive hinge fork
```

The interpretive hinge fork should be visually explicit:

```text
Alternative attestation hinge:
"sufficiently established"

          ┌─ Mandatory reading
          │   sufficient evidence -> approval -> registration path
          │   supports government path
          │
          └─ Discretionary reading
              official determination -> possible denial risk
              supports challenger path
```

### Layer 5: Representative Theorems and Lemmas

Include the four axiom-free structural theorems:

```text
registered-implies-prior-acceptance-path
denied-implies-prior-denial-path
terminal-state-remains-terminal-under-run-trace
all-nonqualifying-implies-no-documentary-proof
```

These should be marked:

```text
0 AXIOMS
AXIOM-FREE STRUCTURAL THEOREM
```

Plain-English explanations:

```text
registered-implies-prior-acceptance-path:
  Registration requires a prior acceptance path.

denied-implies-prior-denial-path:
  Denial requires a prior denial path.

terminal-state-remains-terminal-under-run-trace:
  Once registered or denied, later events do not change the terminal outcome.

all-nonqualifying-implies-no-documentary-proof:
  A bundle of only nonqualifying documents cannot satisfy documentary proof.
```

Also include legal-consequence theorems that depend on assumptions:

```text
conflict-condition-pivots-on-valid-regulation
full-burden-chain
invalid-regulation-enables-conflict-condition
valid-regulation-negates-conflict-condition
```

These should **not** be marked `0 AXIOMS`.

They should be marked:

```text
DEPENDS ON ASSUMPTIONS
```

Important path correction:

```text
full-burden-chain supports the challenger conflict path.
It should not directly support the government no-conflict path.

The government path should instead depend on:
  government-burden-not-severe
  adequate alternative process
  election integrity interest
  reasonable / evenhanded requirement
  valid-regulationp
```

If showing government interaction with the burden chain, use a dashed contested arrow labeled:

```text
Government disputes / negates burden chain
```

Do not use a normal dependency arrow from full-burden-chain to government-model-no-conflict.

### Layer 6: Final Conditional Conclusions

Include:

```text
Challenger model: CONFLICT
  formal conflict condition is established
  conditional on high-risk assumptions, discretionary hinge, and invalid-regulation theory

Government model: NO CONFLICT
  no formal conflict condition
  conditional on government assumptions, mandatory hinge, adequate alternative, and valid-regulation theory
```

Include the central pivot:

```text
Formal pivot:
valid-regulationp
```

The graph should make clear:

```text
If valid-regulationp is true:
  government no-conflict path is supported.

If valid-regulationp is false:
  challenger conflict path is supported, assuming the other preconditions are present.
```

## Hypothetical Controls

The left panel of the web explorer should have user-selectable assumptions.

Use checkboxes or toggle buttons.

### Empirical Assumptions

```text
[ ] Citizen lacks documents through no fault
[ ] Obtaining qualifying documents is materially burdensome
[ ] Government burden-not-severe premise
[ ] Adequate alternative process exists
```

### Interpretive Assumptions

```text
[ ] Mandatory alternative-process reading
[ ] Discretionary alternative-process reading
```

These two should be mutually exclusive unless the user selects a special “compare both” mode.

If both are selected accidentally, show a warning:

```text
The mandatory and discretionary readings are competing interpretations.
The model treats them as separate conditional paths.
```

### Doctrinal Assumptions

```text
[ ] Election integrity is an important government interest
[ ] Documentary proof requirement is reasonable / rationally connected
[ ] Registration procedure is evenhanded
[ ] Severe burden defeats regulation
```

### Structural Filters

```text
[ ] Show axiom-free theorems
[ ] Show high-risk assumptions
[ ] Show ACE trace nodes
[ ] Show only challenger path
[ ] Show only government path
[ ] Show contested premises
```

## Dynamic Behavior

The web app should not run ACL2.

Instead, it should evaluate the dependency graph statically.

When a user toggles assumptions:

```text
Nodes depending on unchecked assumptions should be visually dimmed.
Final conclusions should change status:
  Supported under selected assumptions
  Unsupported under selected assumptions
  Contested / unresolved
```

Example behavior:

### Scenario A

User unchecks:

```text
Obtaining qualifying documents is materially burdensome
```

Then the app should dim:

```text
full-burden-chain
severe-burden derivation
invalid-regulation-enables-conflict-condition
challenger-model-finds-conflict
```

and show:

```text
Challenger conflict path is unsupported under current selected assumptions.
```

### Scenario B

User selects:

```text
Mandatory alternative-process reading
Adequate alternative process exists
Government burden-not-severe premise
Election integrity interest
Reasonable / evenhanded requirement
```

Then the app should highlight:

```text
valid-regulation-negates-conflict-condition
government-model-no-conflict
```

and show:

```text
Government no-conflict path is supported under selected assumptions.
```

### Scenario C

User selects:

```text
Discretionary alternative-process reading
Citizen lacks documents through no fault
Material burden
Alternative process denied / inadequate
```

Then the app should highlight:

```text
full-burden-chain
invalid-regulation-enables-conflict-condition
challenger-model-finds-conflict
```

and show:

```text
Challenger conflict path is supported under selected assumptions.
```

### `tools/serve_explorer.py`

Implement a simple server:

```python
from http.server import ThreadingHTTPServer, SimpleHTTPRequestHandler
from pathlib import Path
import os
import subprocess
import sys

ROOT = Path(__file__).resolve().parents[1]
WEB = ROOT / "web"

def main():
    build_script = ROOT / "tools" / "build_explorer_data.py"
    if build_script.exists():
        subprocess.check_call([sys.executable, str(build_script)], cwd=ROOT)

    os.chdir(WEB)
    server = ThreadingHTTPServer(("127.0.0.1", 8000), SimpleHTTPRequestHandler)
    print("Serving Federal SAVE Act Computational Amicus Explorer at http://127.0.0.1:8000")
    server.serve_forever()

if __name__ == "__main__":
    main()
```
